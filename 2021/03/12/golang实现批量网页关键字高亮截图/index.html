golang实现批量网页关键字高亮截图

Posted on 2021-03-12 Edited on 2021-03-19 In Golang Views: Valine:

前言

老板提出需求说要批量检查URL，高亮其中的敏感字并保存截图，于是我不眠不休加班加点的赶出了v1.0版本，当然也踩好多坑，在此记录一下，v1.0是单线程版。

v2.0是针对excel格式的并发版，抓取第三列的所有链接，图片的保存名称是第一列的序号，方便对照查找，高并发对性能有要求，正常开20-40就够了。

点这里下载

说明

单线程版本源码分了两个文件，后期便于维护，无法直接go run运行，需要go build生成二进制文件，然后双击运行二进制文件即可，如果担心错过详细的输出信息，可以在命令行中运行，关于关键字高亮的实现方式是使用Chromedp包内嵌JS执行。

有关源码

源码分了两个包，一个是主包main.go，另外一个targetFile.go负责对读取目标文件并进行正则匹配以及格式转换，最终返回一个格式正确无重复记录的文件句柄，targetFile包中的所有函数都被RegexpHTTP函数调用，可以理解成一个整体，为后继的并发版本体提供了良好的开发基础。

使用说明

  • 运行程序
  • 输入需要过滤的敏感词，如果有多个词组，每个词组之间使用竖线|分隔
  • 设置页面加载时长，假设页面默认等待加载时长设定为5秒，由于需要手机端截图+PC端截图，执行程序时每个URL就是10秒
  • 拖拽存放大量需要检测的url文件到其中，需要txt格式。如果您是excel文件，打开excel文件按下ctrl+a组合键全选后复制粘贴到一个txt文件中即可放入自动检测筛选url
  • txt中每个超链接之间需要被分隔，分隔符可以是空格或者换行，其他无要求，以便于程序自动搜索匹配超链接
  • 程序还会将匹配到的超链接保存在原链接文件的相同路径下，重命名为***筛选去重.txt
  • 程序会逐条访问***筛选去重.txt中的链接并高亮关键字后截图保存，截图保存在本程序同级目录下的当天日期的文件夹内
  • 同时存在两个文件夹，一个存放pc端截图，另一个存放手机端截图
  • 对于无法访问的目标同样会在程序相同路径下创建文件存放无法访问的url，文件名为errorHTTP.txt

通道问题

for range遍历通道

for range循环可以遍历通道，当通道Close后遍历结束，如果通道没有Close则报错deadlock。

select遍历通道

select从已关闭通道中读取for 配合 select 同样可以遍历通道，但是即使通道已经关闭还是可以取出，string类型的通道则不断取出空string，int类型则不断取出0，并不会执行default中的内容。缩放复制select从未设置Close的通道中读取但是如果通道不关闭，则所有值取出后select阻塞，并不会默认执行default中的内容，需要设置超时跳出。

通道小结

推荐使用for range循环遍历通道，通道一定记得关闭。

select主要用于设置通道超时关闭，比如：

select{

case <- chan:

    ......

case <-time.After(time.Second*3):

    fmt.Print(“通道超时关闭”)

    return

}

default用的很少，一般都是同时生产同时消费，难免出现时间差，此时使用default判断就很容易导致通道还在使用，消费者就提前退出了。
